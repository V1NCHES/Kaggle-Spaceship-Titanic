# Spaceship Titanic: Пошаговое решение ML задачи

Этот репозиторий содержит решение задачи **Spaceship Titanic** с платформы Kaggle.

## 🚀 Описание задачи

**Spaceship Titanic** — это соревнование по машинному обучению, вдохновленное классической задачей Титаника, но в сеттинге будущего.

**Сюжет:** Космический корабль "Титаник" столкнулся с пространственной аномалией. Почти половина пассажиров была перенесена в другое измерение. Наша задача — предсказать, какие пассажиры были перенесены (`Transported`), на основе данных об их полете, возрасте, планете и тратах на борту.

**Источник:** [Kaggle Spaceship Titanic Competition](https://www.kaggle.com/competitions/spaceship-titanic)

---

## 🛠 Что было сделано

В этом проекте реализован полный цикл разработки ML-модели:

1.  **Настройка окружения**: Исправлены проблемы совместимости библиотек (`numpy`, `pandas`) для стабильной работы.
2.  **Исследовательский анализ данных (EDA)**:
    *   Проверен баланс классов (целевая переменная распределена почти 50/50).
    *   Выявлены ключевые зависимости: например, пассажиры в состоянии криосна (`CryoSleep`) имели гораздо больше шансов на перемещение.
3.  **Feature Engineering (Генерация признаков)**:
    *   Разбор `PassengerId`: извлечена информация о группах пассажиров.
    *   Разбор `Cabin`: выделены палуба (`Deck`), номер кабины (`Num`) и сторона корабля (`Side` — Port/Starboard).
    *   Создан суммарный признак `TotalSpending` (общие траты на услуги корабля).
4.  **Умная обработка пропусков**:
    *   Вместо удаления строк или простого заполнения средним, использована логическая связь. Например, если пассажир в криосне, его траты автоматически равны 0. Планеты восстанавливались по номеру группы семьи.
5.  **Построение и сравнение моделей**:
    *   Сравнили работу разных алгоритмов: `Logistic Regression`, `Decision Tree`, `Random Forest` и `CatBoost`.
    *   Использована кросс-валидация (`Stratified K-Fold`) для получения честной оценки точности.
6.  **Интерпретация модели (SHAP)**:
    *   Использованы графики SHAP для «заглядывания в мозг» модели. Это позволило увидеть, как именно каждый признак (например, возраст или палуба) влияет на финальное решение.
7.  **Экспериментальный подход**:
    *   Протестировано решение задачи через регрессию (`Ridge`, `Lasso`) для оценки «уверенности» модели.
8.  **Финальное предсказание**:
    *   Лучшая модель (`CatBoost`) использована для генерации финального файла `final_submission.csv`.

## 📈 Результаты
Модель градиентного бустинга показала наилучшие результаты, эффективно обрабатывая категориальные признаки и пропуски в данных.
